# User Guide: aggregateGenCodeDesc

Welcome to the User Guide for `aggregateGenCodeDesc`. This tool tracks and aggregates the exact percentage of AI-generated code over a given repository time window.

## Quick Start
The fastest way to use the tool is to run it directly inside your local Git repository using Algorithm A (Live Blame):

```bash
# 1. CD into your target repository
cd /path/to/your/git/project

# 2. Run the tool
python aggregateGenCodeDesc.py \
  --repoURL $(git config --get remote.origin.url) \
  --repoBranch main \
  --startTime 2026-03-01T00:00:00Z \
  --endTime 2026-03-31T23:59:59Z \
  --genCodeDescDir /path/to/gen_code_data \
  --alg A \
  --threshold 60
```

## Overview
`aggregateGenCodeDesc` analyzes code surviving at a specific snapshot and calculates AI involvement using three distinct metrics:
1. **Weighted Mode**: Sum of `(genRatio / 100)` divided by the total surviving code lines.
2. **Fully AI Mode**: Percentage of lines completely generated by AI (`genRatio == 100`).
3. **Mostly AI Mode**: Percentage of lines predominantly generated by AI (`genRatio >= threshold`).

## Usage Guide

```bash
python aggregateGenCodeDesc.py \
  --repoURL <URL_KEY> \
  --repoBranch <BRANCH_NAME> \
  --startTime <ISO8601_START_TIME> \
  --endTime <ISO8601_END_TIME> \
  --genCodeDescDir <DIR_PATH> \
  [--alg <A|B|C>] \
  [--patchesDir <DIR_PATH>] \
  [--threshold <0-100>] \
  [--log-level <DEBUG|INFO|WARN|ERROR>]
```

### Required Parameters
- `--repoURL`: The repository's identity URL (e.g., `https://github.com/EnigmaWU/project`). **Crucial:** Because this is used as a lookup validation key, it *must* strictly match the `REPOSITORY.repoURL` field recorded inside your `genCodeDesc` metadata files.
- `--repoBranch`: The branch (for Git) or path (for SVN) to analyze.
- `--startTime`: The start of your time window (ISO 8601, e.g., `2026-01-01T00:00:00Z`).
- `--endTime`: The end of your time window.
- `--genCodeDescDir`: The path to the directory where your per-revision `genCodeDesc` JSON metadata files are stored.

### Optional Parameters
- `--alg`: The analysis algorithm to deploy (`A`, `B`, or `C`). Defaults to `A`.
  - `A` (Live Blame): Extremely accurate. Requires you to run the script from inside the repository checkout so it can execute live `git blame` or `svn blame`.
  - `B` (Offline Diff): Offline calculation. Replays diff changes incrementally. Requires `--patchesDir`.
  - `C` (Embedded Blame): Fully offline, zero VCS access requirements. Consumes `v26.04` metadata purely.
- `--patchesDir`: **Required only for `--alg B`**. Points to the directory containing pre-fetched diff artifacts (`localCommitPatches`).
- `--threshold`: The percentage threshold for the "Mostly AI" metric (default is `60`).
- `--log-level`: Diagnostic log verbosity. Options are `DEBUG`, `INFO`, `WARN`, `ERROR` (default is `INFO`).

## Detailed Examples by Algorithm

### 1. Algorithm A: Live Blame (Git/SVN)
AlgA queries the code repository directly. You must execute this command from within the localized repository checkout directory.
```bash
cd /path/to/your/git_or_svn_checkout
python aggregateGenCodeDesc.py \
  --repoURL https://github.com/EnigmaWU/MyProject \
  --repoBranch develop \
  --alg A \
  --startTime 2026-01-01T00:00:00Z \
  --endTime 2026-04-01T00:00:00Z \
  --genCodeDescDir ./my_metadata
```

### 2. Algorithm B: Offline Diff Replay (Git/SVN)
AlgB operates offline. Since it doesn't run live blame, you can safely execute it from anywhere. You simply provide the metadata directory and the commit patches directory.
```bash
python aggregateGenCodeDesc.py \
  --repoURL https://svn.example.com/repo \
  --repoBranch /trunk \
  --alg B \
  --startTime 2026-02-01T00:00:00Z \
  --endTime 2026-02-28T23:59:59Z \
  --genCodeDescDir /offline/svn_metadata \
  --patchesDir /offline/svn_localCommitPatches
```

### 3. Algorithm C: Embedded Blame (v26.03+)
AlgC linearly processes self-sufficient `v26.03+` metadata files. It needs absolute zero VCS access—no local patches and no checkouts. It just accumulates surviving lines via embedded metadata mapping.
```bash
python aggregateGenCodeDesc.py \
  --repoURL https://gitlab.internal/core/auth \
  --repoBranch main \
  --alg C \
  --startTime 2026-01-01T00:00:00Z \
  --endTime 2026-04-19T00:00:00Z \
  --genCodeDescDir /offline/v2604_metadata \
  --threshold 80
```

## Output Format
The tool prints a JSON object to standard output (`stdout`) matching the `genCodeDesc` v26.03 protocol schema. The aggregate metric values you requested are embedded cleanly inside the `SUMMARY` block.

```jsonc
{
  "protocolName": "generatedTextDesc",
  "protocolVersion": "26.03",
  "SUMMARY": {
    "totalLines": 5000,
    "weightedModeRatio": 42.5,
    "fullyAIModeRatio": 30.0,
    "mostlyAIModeRatio": 52.0
  },
  "DETAIL": [
    // Array of processed files with their surviving AI-attributed code lines
  ],
  "REPOSITORY": {
    "vcsType": "git",
    "repoURL": "https://github.com/EnigmaWU/MyGenCodeDescBase",
    "repoBranch": "main",
    "startTime": "2026-03-01T00:00:00Z",
    "endTime": "2026-03-31T23:59:59Z"
  }
}
```

*Note: All runtime logs (like those from `--log-level`) are routed to standard error (`stderr`) so they will not interfere with capturing or piping the JSON data.*
